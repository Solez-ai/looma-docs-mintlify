This guide walks you through cloning Looma, installing its dependencies, and launching the editor in your browser. Because Looma is fully client-side, there is no environment configuration, database setup, or API key required to get the editor working locally.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 20 or later** — Looma uses features available in the Node.js 20 LTS release. You can check your version with `node -v`.
- **npm** — comes bundled with Node.js. Looma uses npm for dependency management.

## Running locally

<Steps>
  <Step title="Clone the repository">
    Clone the Looma source from GitHub and move into the project directory.

    ```bash
    git clone https://github.com/Solez-ai/looma.git
    cd looma
    ```
  </Step>
  <Step title="Install dependencies">
    Install all Node.js dependencies declared in `package.json`. This includes Next.js, Three.js, FFmpeg.wasm, and all supporting packages.

    ```bash
    npm install
    ```

    Expect the install to take a minute or two on first run — FFmpeg.wasm includes WebAssembly binaries that are larger than typical npm packages.
  </Step>
  <Step title="Start the development server">
    Start the Next.js development server with hot-module reloading enabled.

    ```bash
    npm run dev
    ```

    You should see output similar to:

    ```
    ▲ Next.js 16
      - Local:        http://localhost:3000
      - Environments: .env.local
    
    ✓ Starting...
    ✓ Ready in 2.1s
    ```
  </Step>
  <Step title="Open the editor">
    Open [http://localhost:3000](http://localhost:3000) in your browser. The Looma editor loads immediately. No login screen, no onboarding flow — the editor is the landing page.

    To open the video editor directly, navigate to:

    ```
    http://localhost:3000/en?mode=video
    ```

    To open the photo editor:

    ```
    http://localhost:3000/en?mode=photo
    ```
  </Step>
</Steps>

## Production build

When you are ready to deploy or benchmark a production bundle, use the standard Next.js build and start commands.

```bash
npm run build
npm start
```

`npm run build` compiles and statically optimises the application. `npm start` serves the compiled output on port 3000. Because Looma has no server-side logic, the production build can also be exported as a fully static site.

<CodeGroup>
  ```bash Development
  # Start the dev server with hot reloading
  npm run dev
  ```

  ```bash Production
  # Build an optimised production bundle, then serve it
  npm run build
  npm start
  ```
</CodeGroup>

<Tip>
  Because Looma is fully client-side, the production build is a static Next.js export. You can host it on any static file server — Vercel, Cloudflare Pages, GitHub Pages, or a plain nginx container — with no Node.js runtime required at the server level. All video encoding and media processing continues to run in the visitor's browser via FFmpeg.wasm.
</Tip>
