Looma's zoom system is built around time-ranged **fragments** — discrete intervals on the video timeline where the canvas zooms in, holds, and zooms back out. Each fragment is self-contained: it carries its own zoom level, transition speed, focus point, optional camera movement, and optional 3D tilt effect. All zoom types and utilities live in `types/zoom.types.ts` and are re-exported from `@/types`.

```typescript
import type { ZoomFragment, ZoomPhaseState } from '@/types';
import {
  calculateZoomPhaseState,
  createZoomFragment,
  generateDefaultZoomFragments,
  zoomLevelToFactor,
  speedToTransitionMs,
  formatZoomTime,
  ZOOM_EASING,
} from '@/types';
```

---

## `ZoomFragment`

A `ZoomFragment` represents a single zoom event on the timeline. The fragment spans from `startTime` to `endTime`, and is internally divided into three phases: **entry** (zoom in), **hold** (maintain zoom), and **exit** (zoom out). The hold duration is what remains after the two transition periods are subtracted from the total duration.

```typescript
interface ZoomFragment {
  /** Unique identifier — generated by createZoomFragment(). */
  id: string;

  /** Start of the zoom event in seconds. */
  startTime: number;

  /** End of the zoom event in seconds. */
  endTime: number;

  /** Integer zoom level 1–10; converted to a CSS scale factor by zoomLevelToFactor(). */
  zoomLevel: number;

  /** Integer speed 1–10; controls entry/exit transition duration via speedToTransitionMs(). */
  speed: number;

  /** Horizontal focus point as a percentage (0–100) of the video width. */
  focusX: number;

  /** Vertical focus point as a percentage (0–100) of the video height. */
  focusY: number;

  // --- Camera movement (optional) ---

  /** When true, the focus point animates from (focusX, focusY) to (movementEndX, movementEndY)
   *  during the hold phase. */
  movementEnabled?: boolean;

  /** Target horizontal focus point for camera movement (0–100). Defaults to focusX. */
  movementEndX?: number;

  /** Target vertical focus point for camera movement (0–100). Defaults to focusY. */
  movementEndY?: number;

  /** Seconds after the hold phase starts before camera movement begins. Defaults to 0. */
  movementStartOffset?: number;

  /** Seconds after the hold phase starts when camera movement ends. Defaults to holdDuration. */
  movementEndOffset?: number;

  // --- 3D perspective tilt (optional) ---

  /** Enables the 3D CSS perspective effect during the zoom. */
  enable3D?: boolean;

  /** Intensity of the 3D effect as a percentage (0–100). */
  perspective3DIntensity?: number;

  /** Base rotateX angle for the 3D tilt, in degrees (-45 to 45). */
  perspective3DAngleX?: number;

  /** Base rotateY angle for the 3D tilt, in degrees (-45 to 45). */
  perspective3DAngleY?: number;
}
```

## `ZoomPhaseState`

`ZoomPhaseState` is the computed, per-frame output of `calculateZoomPhaseState()`. It tells the renderer exactly what CSS transforms to apply at any given moment in the timeline.

```typescript
interface ZoomPhaseState {
  /** Which of the three zoom phases is currently active. */
  phase: 'entry' | 'hold' | 'exit';

  /** CSS scale factor to apply to the video layer. */
  scale: number;

  /** Horizontal pan offset as a percentage (0–100) of the video width. */
  focusX: number;

  /** Vertical pan offset as a percentage (0–100) of the video height. */
  focusY: number;

  /** Progress through the current phase, from 0 (start) to 1 (end). */
  progress: number;

  /** CSS rotateX degrees for the 3D tilt effect (0 when 3D is disabled). */
  rotateX: number;

  /** CSS rotateY degrees for the 3D tilt effect (0 when 3D is disabled). */
  rotateY: number;

  /** CSS perspective in pixels. Either 0 (no 3D) or 500 (3D active). */
  perspective: number;
}
```

## Easing Functions

Two easing functions shape the zoom animation curves. They accept a normalized time value `t` in the range `[0, 1]` and return an eased value in the same range.

```typescript
/** Quartic ease-out. Applied to zoom entry and exit scale transitions.
 *  Starts fast and decelerates — gives snappy, professional-feeling zooms. */
function easeOutQuart(t: number): number {
  return 1 - Math.pow(1 - t, 4);
}

/** Quartic ease-in-out. Applied to camera movement during the hold phase.
 *  Accelerates then decelerates — smooth pan between two focus points. */
function easeInOutQuart(t: number): number {
  return t < 0.5 ? 8 * t * t * t * t : 1 - Math.pow(-2 * t + 2, 4) / 2;
}
```

## Utility Functions

| Function | Signature | Returns |
|----------|-----------|---------|
| `calculateZoomPhaseState` | `(fragment: ZoomFragment, currentTime: number, forExport?: boolean)` | `ZoomPhaseState` |
| `createZoomFragment` | `(startTime: number, endTime: number)` | `ZoomFragment` |
| `generateDefaultZoomFragments` | `(videoDuration: number)` | `ZoomFragment[]` |
| `calculateHoldDuration` | `(fragment: ZoomFragment)` | `number` (seconds) |
| `zoomLevelToFactor` | `(level: number)` | `number` (1.2 – 4.0) |
| `speedToTransitionMs` | `(speed: number)` | `number` (150 – 2000 ms) |
| `formatZoomTime` | `(seconds: number)` | `string` (`"MM:SS"`) |

### `calculateZoomPhaseState`

The core per-frame computation. Call this every time the video `currentTime` changes to get the transforms you need to apply to the canvas.

```typescript
// In a render loop or React effect:
const phaseState = calculateZoomPhaseState(fragment, video.currentTime);

videoLayerEl.style.transform = `
  scale(${phaseState.scale})
  rotateX(${phaseState.rotateX}deg)
  rotateY(${phaseState.rotateY}deg)
`;
videoLayerEl.style.perspective = phaseState.perspective
  ? `${phaseState.perspective}px`
  : 'none';
```

Pass `forExport = true` when rendering to an offscreen canvas. In export mode the function returns exact intermediate scale values for each frame rather than relying on CSS transitions.

### `createZoomFragment`

Creates a new fragment with safe defaults: `zoomLevel = 1.5`, `speed = 5`, `focusX = 50`, `focusY = 50`, movement disabled.

```typescript
const fragment = createZoomFragment(2.5, 6.0);
// → { id: 'zoom_...', startTime: 2.5, endTime: 6.0, zoomLevel: 1.5, speed: 5, ... }
```

### `generateDefaultZoomFragments`

Generates two evenly-spaced starter fragments across the full video duration. Useful when a video first loads and the user has no existing fragments.

```typescript
const fragments = generateDefaultZoomFragments(30); // 30-second video
// → two ZoomFragment objects at ~5s and ~20s
```

### `calculateHoldDuration`

Returns how many seconds the hold phase lasts after both transitions are subtracted from the fragment's total duration. Use this to validate whether camera movement offsets fit within the available window.

```typescript
const holdSeconds = calculateHoldDuration(fragment);
if (fragment.movementEndOffset > holdSeconds) {
  console.warn('Movement end offset exceeds hold duration');
}
```

### `zoomLevelToFactor`

Converts the integer `zoomLevel` (1–10) to the actual CSS `scale()` factor (1.2–4.0) via linear interpolation.

```typescript
zoomLevelToFactor(1);  // → 1.2  (minimal zoom)
zoomLevelToFactor(5);  // → 2.6  (medium zoom)
zoomLevelToFactor(10); // → 4.0  (maximum zoom)
```

### `speedToTransitionMs`

Converts the integer `speed` (1–10) to a transition duration in milliseconds (150–2000 ms). Higher speed = shorter transition = snappier zoom.

```typescript
speedToTransitionMs(1);  // → 2000 ms (slowest — cinematic)
speedToTransitionMs(5);  // → 1081 ms (medium)
speedToTransitionMs(10); // → 150  ms (fastest — instant)
```

### `formatZoomTime`

Formats a raw seconds value as a zero-padded `"MM:SS"` string for display in the zoom editor timeline.

```typescript
formatZoomTime(0);    // → "00:00"
formatZoomTime(75);   // → "01:15"
formatZoomTime(3600); // → "60:00"
```
